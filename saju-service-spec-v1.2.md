## 0. 제품 정의

**한 줄 정의:** 생년월일시를 입력하면 실제 철학관 상담의 리듬(훅→문답→처방)을 재현한 카드형 사주 리포트를 생성하는 서비스. **전 세대의 고민을 받되, 세대별로 다른 표면을 입는다. 바이럴은 웹이, 리텐션은 앱이, 세대 확장은 가족 루프가 담당한다.**

**전략 6원칙**
1. **계산은 코드가, 서사는 AI가** — 만세력 산출은 결정론적 코드(ssaju+어댑터). LLM은 해석 텍스트만.
2. **결과물 = 공유 콘텐츠** — 모든 결과는 카드 슬라이드. 공유 1채널은 **카카오톡**(OG 품질=전환율), 2채널은 인스타 스토리.
3. **경험 = 의식(ritual)** — 뜸(붓글씨 연출), 단정형 훅, 문답, 처방의 상담 시퀀스.
4. **운영 부담 제로 지향** — 서버리스, 회원가입 없음, 해석 캐싱으로 비용-트래픽 디커플링.
5. **하나의 엔진, 세대 적응형 표면** — 중간 어딘가의 애매한 제품 금지. 톤·UI·질문 노출만 세대별 분기, 엔진·카드 구조·루프는 공유.
6. **결제 증거를 최대한 앞당긴다** — 8월 베타부터 실결제. "만들면 팔린다" 금지.

**차별화 자산 3층:** ① "겁주지 않는 사주" 포지셔닝(경쟁 앱 1점 리뷰의 역상) ② 배포 속도·채널 축적 ③ **일간 10종 캐릭터 IP** (포맷은 카피돼도 캐릭터는 카피 불가 — v1부터 IP 1호로 취급, 카카오 이모티콘은 v2 후보).

## 1. MVP 범위

### In (v1.0 출시)
- [공통] 대화체 입력 → 만세력 산출 → 역술 시퀀스 카드 리포트 (8장, MZ/클래식 모드)
- [공통] 전 세대 고민 카탈로그 + 연령별 적응 노출 (섹션 7)
- [웹] 공유 OG(카톡 최적화), 궁합·가족 루프, 단건 결제, **선물하기**
- [앱] 동일 리포트 + 데일리 운세 푸시 + 구독(IAP)
- [유료] 복채 문답, 버티컬 리포트(자녀운·택일·시험운·이직/결혼 타이밍)

### Out (만들지 않음 / 보류)
- 회원가입, 채팅 상담, 관상·손금·성명학, 역술인 중개 마켓플레이스(운영 부담 원칙 충돌 — v2에도 신중)
- 토정비결 모듈(v1.5 — 12월 전 완성 목표), TTS 음성 리포트(v2 — 기성세대 접근성 관점에서 우선순위 상향됨), 데일리 체크인(v2)

- ssaju 버전 고정 + 업데이트 시 회귀 테스트 전체 통과를 CI 조건으로.

## 3. 만세력 엔진 스펙 (v1.1 규칙 유지 + 어댑터 반영)

입력: `{ birthDate, birthTime?, gender?, calendarType, birthRegion?, unknownTime }`
출력 `SajuChart`: 4주 간지 · 오행 분포 · 십신 · 대운 · 지장간 · 12운성 · 격국/용신(유료) · **경계출생 플래그**

규칙: ① 연주=입춘 절입시각 ② 월주=12절 절입시각 ③ 일주=60갑자 순환 ④ 시주=시두법 ⑤ 진태양시=출생지 경도 기준(regions.ts, 기본 -30분) ⑥ DST 보정(어댑터) ⑦ 조자시설 기본(상수 전환 가능) ⑧ 출생시 모름=3주 폴백 ⑨ 표준시 변경기 이중보정 차단(어댑터).

**택일 모듈(taekil.ts, 신설):** 목적(이사/개업/계약/행사) + 기간 입력 → 해당 기간 일진 스캔 → 본인 일간과의 관계(합·충·공망 회피 등 규칙 테이블)로 "좋은 날 3일 + 피할 날" 산출. 순수 계산이라 LLM 비용 0, 결과 문구만 캐시 유닛.

## 4. AI 해석 레이어

### 4.1 파이프라인 (유지)
`SajuChart → 해석 유닛 분해 → 캐시 조회 → (미스) LLM 생성 → guardrails → 조립`
- LLM에 생년월일 원본 미전달. 캐시 키 = `유닛:값:프롬프트버전:시즌:톤모드:관심사`

### 4.2 톤 축 확장 (v1.2)
캐시 키에 **mode: 'mz' | 'classic'** 추가.
- **MZ 모드:** 힙한 단정형, 반박 여지 없는 직설 ("사람들이 당신한테 기대죠. 근데 정작 당신이 기댈 데는 없고.")
- **클래식 모드:** 정중한 상담체, "선생님이 봐주는" 격식 ("주변에서 많이 의지하는 사주입니다. 정작 본인 마음 둘 곳은 잘 안 보이셨을 테고요.")
- 기본값: 생년 기준 자동(만 40세 경계) + 결과 화면에서 수동 전환 토글. 반응 분기(소름/글쎄요) 톤 2종과 조합 → 톤 변형 총 4종 (조합 폭발 아님 — 유닛 단위 캐시).

### 4.3 가드레일 — 카테고리별 3단계 (v1.2 강화)
| 레벨 | 카테고리 | 규칙 |
|---|---|---|
| L1 공통 | 전체 | 건강 진단·사고 예언·투자 지시·사망/이혼 단정·공포 소구·부적/굿 등 지출 유도 금지. 하단 고지문 상시 |
| L2 관망 언어 | 부동산·사업·재물 시기 | "사라/팔라" 금지 → "서두르기보다 살펴보기 좋은 시기" 수준 관망 표현만 (자문업 오해 차단) |
| L3 최고 민감 | 자녀운·건강한 한 해 | 불안 자극 표현("이대로면 어렵다") 금지. 건강은 "쉬어갈 시기 안내" 프레임만. 클래식 모드에서 고지문 강화(기성세대는 의사결정 근거로 수용하는 경향) |
- guardrails.ts에 카테고리별 금지 패턴 테이블 + 후처리 필터. 프롬프트에도 카테고리별 지시 삽입.

## 5. 경험 설계 — 역술 시퀀스 (v1.1 유지)

① 대화체 입력("태어난 시간, 확실하세요?") → ② 붓글씨 팔자 애니메이션(2~3초 뜸, 스킵 가능) → ③ 일주 60종 단정형 훅(타이핑 연출) → ④ 원탭 반응 분기("어때요, 맞나요?") → ⑤ "복채 내고 하나 묻기"(유료 문답) → ⑥ 개운 처방전(=공유 카드) → ⑦ 토큰 재방문.
비주얼: 한지·먹빛·주홍 낙관 포인트를 모던 레이아웃 위에. **클래식 모드 UI:** 글씨 최소 18pt급, 카드당 정보량 축소, 스와이프+큼직한 [다음] 버튼 병행, 대비 강화.

## 6. 리포트 카드 구성 (8장, v1.1 유지 + 모드 분기)

0 팔자 연출 → 1 단정형 훅 → 2 나의 팔자(오행 SVG 차트) → 3 타입 카드(일간 10종 IP) → [반응 체크] → 4 성격 코어 → [고민 문답 — 섹션 7의 적응 노출] → 5 하반기 운(선택 고민 중심) → 6 조심할 것 티저(블러) → 7 개운 처방전(공유 겸용, 카톡 OG 1순위 디자인) → 8 CTA 허브 [복채 내기 / 궁합·가족 보기 / 선물하기 / 매일 아침 운세(앱)]

## 7. 질문 체계 — 전 세대 고민 카탈로그 (v1.2 핵심 신설)

### 7.1 원칙
관심사 4택(연애/커리어/재물/인간관계)은 20~30대 편향이었음 → **전 세대 카탈로그 + 연령별 적응 노출**로 교체. 생년월일을 이미 알므로 별도 설정 없이 자동 적응.

### 7.2 카탈로그 (core/content/concerns.ts)
| 영역 | 10~20대 | 20~30대 | 40~50대 | 60대+ |
|---|---|---|---|---|
| 사랑·관계 | 썸·연애, 짝사랑 | 결혼 타이밍, 이 사람인가 | 부부 관계, 자녀 혼사 | 배우자, 가족 화합 |
| 일·돈 | 진로, 시험운 | 취업, 이직, 창업 | 사업 확장/정리, 승진, 은퇴 준비 | 노후 재정 |
| 재물·거처 | (라이트) | 목돈 시기, 독립·이사 | 부동산 매매 시기, 이사 방위 | 정리와 안정 |
| 가족 | 부모와의 갈등 | 부모 걱정 | **자녀운(수능·취업·결혼)** | 자식·손주 |
| 시기·택일 | 시험 날짜 | 계약·퇴사 시점 | **이사·개업·혼사 택일** | 행사 택일 |
| 건강 | — | — | 건강한 한 해 (L3 프레임) | 건강한 한 해 (L3) |

데이터 스키마: `{ id, label, ageWeights, promptTemplate, guardrailLevel, verticalSku? }`

### 7.3 UX
- "요즘 뭐가 제일 답답해요?" 화면: **연령대별 기본 4개 노출** (28세: 이직/연애/목돈/인간관계 · 55세: 자녀/사업·은퇴/이사·매매/건강한 한 해) + "다른 고민 보기"로 전체 카탈로그 (연령 밖 선택 자유 보장)
- 복채 문답 자유 입력 플레이스홀더도 연령별 분기 ("연말 이직 제안이 고민이에요" vs "아들 결혼이 자꾸 늦어지는데요")

### 7.4 카탈로그 = SKU 매트릭스 (버티컬 유료 상품)

## 8. 확산 루프 — 궁합 + 가족 + 선물 (v1.2 확장)

### 8.1 궁합·가족 루프
관계 유형: 연인/썸/친구/직장 + **부모-자녀/부부/형제** (가족 궁합). 초대 수신자는 무조건 웹(설치 요구 금지). 나쁜 결과 없음 원칙 유지(불꽃형=재미 프레임).
**가족 루프 = 세대 확장의 유통 구조:** 자녀→부모("엄마 사주 봐드렸어", 설·추석·어버이날 트리거) / 부모→자녀(자녀운 SKU) / 명절 가족 궁합 콘텐츠.

### 8.2 선물하기 (v1.2 신설, Phase 5 +1일)
결제 시 [나에게/선물하기] 분기 → 수신자용 언락 링크 + 선물 포장 연출 카드(보내는 사람 메시지 1줄) → 수신자는 웹에서 즉시 열람 → 신규 유저 전환. **유료 바이럴 루프.** 생일·명절·수능 D-day가 천연 결제 트리거. 방향: 친구↔친구, 자녀↔부모 양방향.

### 8.3 계측
퍼널: `input_start → report_view → reaction_tap → share_click(채널 구분: kakao/insta/save) → compat_invite_* → gift_sent → gift_opened → paywall_view → purchase(sku 구분) → app_install_click → push_optin`
북극성 = K-factor. 보조 = SKU별 전환율, 선물 오픈→신규 리포트 전환율.
