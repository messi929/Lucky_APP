# ssaju 라이브러리 검증 리포트 (2026-07-12)

## 검증 방법
삼중 교차: **ssaju**(채택 후보) vs **lunar-javascript**(중국계 독립 구현, 천문 계산) vs **60갑자 앵커 산술**(일주 독립 검증, 1900-01-01=甲戌 기준). 케이스 16종: 평범 5 + 입춘 경계 4 + 월 절기 경계 2 + 자시 경계 3 + 서머타임 2 (+음력 변환, 절입 분 단위 스캔, 표준시 변경기 추가 확인).

## 결과 요약: **조건부 채택 (어댑터 레이어 필수)**

### ✅ 통과 항목
| 항목 | 결과 |
|---|---|
| 연주·월주 (절기 경계 포함) | lunar-js와 전 케이스 일치. 1990/2000 입춘 직전·직후, 입동 직전·직후 모두 정확히 갈림 |
| 일주 | 앵커 산술과 16/16 일치 (자정 경계 포함) |
| 시주 (시두법) | 전 케이스 정확 |
| 음력 입력 (calendar:'lunar') | 음력 1990-04-21 → 양력 1990-05-15와 팔자 완전 일치 |
| 진태양시 보정 (applyLocalMeanTime) | 서울 15:10 → 보정 후 未시 전환 확인. 정상 작동 |
| 자시 처리 | 자정 기준 일주 변경(조자시설) — **기획서 v1.1 기본 방침과 일치** |

### ⚠️ 어댑터에서 해결해야 할 항목 (corrections.ts 책임)
1. **서머타임 미보정 (실측 확인).** 1988-07-15 15:30(DST 벽시계) 입력 시 申시 반환 — 보정했다면 未시여야 함. 1957년도 동일.
   → **해결:** corrections.ts에서 한국 DST 기간(1948–1951, 1955–1960, 1987–1988) 출생 입력을 -1시간 변환 후 ssaju에 전달. (기획서 규칙 6 그대로 — 라이브러리 밖에서 우리가 처리)
2. **절입 시각 ±2분 오차.** 1990 입춘 플립: ssaju 11:12 vs lunar-js 11:14 (KST). ssaju는 근사 공식 사용.
   → **해결(택1):** (a) KASI 절기 테이블(공공데이터)로 절입 ±5분 이내 출생만 "경계 출생" 플래그 → 리포트에 "절입 경계 출생이라 두 명식이 가능합니다" 안내 (역술 관행상 오히려 전문성 어필) / (b) KASI 시각으로 연·월주 오버라이드. **v1은 (a) 권장** — 구현 최소, 신뢰 확보.
3. **표준시 변경기(1954-08~1961-08, UTC+8.5) 이중 보정 위험.** 이 시기는 표준시가 이미 127.5°E라 진태양시 보정이 거의 불필요한데, applyLocalMeanTime을 일괄 적용하면 -32분 이중 보정 발생.
   → **해결:** corrections.ts에서 출생일이 해당 기간이면 LMT 보정 비활성(또는 잔여분만).
4. **야자시 시간(時干) 유파 차이 인지.** 23:00~24:00 시주 간(干)에 대해 ssaju는 당일 일간 기준(癸日→壬子), lunar-js sect2는 익일 간(甲子). 버그가 아닌 유파 차이. ssaju 방식이 조자시설과 정합 → **채택하되 문서화.**

## 아키텍처 반영 (core/saju 구조 변경)
```
core/saju/
  corrections.ts   # [우리 코드] DST 테이블, 표준시 변경기 처리, 경도별 LMT on/off
  engine.ts        # [우리 코드] corrections → ssaju 호출 → SajuChart 매핑
  boundary.ts      # [우리 코드] KASI 절기 테이블로 절입 ±5분 경계 플래그
  (ssaju)          # [외부] 계산 코어. 직접 수정 금지, 버전 고정(pin)
```
- ssaju 버전은 lockfile로 고정하고, 업데이트 시 회귀 테스트 전체 재실행을 CI 조건으로.
- 보너스: ssaju는 십성·12운성·격국·용신·신살·대운·세운·월운까지 산출 + toCompact() LLM 압축 출력 → **유료 리포트 콘텐츠 깊이와 해석 파이프라인 입력이 공짜로 확보됨.** Phase 1 예상 기간 3~5일 → 1~2일로 단축.
