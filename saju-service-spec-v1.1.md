# 사주 카드 리포트 서비스 — Claude Code 개발 기획서 v1.1

> 개정일: 2026-07-12 | v1.0 대비 변경: ①iOS/AOS 앱 추가(모노레포) ②역술 경험 레이어 ③입력 티어 설계 ④레퍼런스 분석 프로세스 ⑤카드 구성 개편
> 사용법: 각 Phase를 순서대로 Claude Code 세션에 투입. Phase 0~1 완료 전 UI 작업 금지.

## 0. 제품 정의

**한 줄 정의:** 생년월일시를 입력하면 실제 철학관 상담의 리듬(훅→문답→처방)을 재현한 카드형 사주 리포트를 생성하고, 궁합 초대 루프로 확산되는 서비스. **바이럴은 웹이, 리텐션은 앱이 담당한다.**

**전략 4원칙 (모든 의사결정 기준)**
1. **계산은 코드가, 서사는 AI가** — 만세력 산출은 100% 결정론적 코드. LLM은 해석 텍스트만.
2. **결과물 = 공유 콘텐츠** — 모든 결과는 카드 슬라이드. 리포트 자체가 광고판.
3. **경험 = 의식(ritual)** — 정보 전달이 아니라 상담 경험의 재현. 뜸(노동 착시), 단정형 훅, 문답, 처방으로 이어지는 시퀀스.
4. **운영 부담 제로 지향** — 서버리스, 회원가입 없음, 비용은 트래픽과 디커플링(해석 캐싱).

**유통 구조 (하이브리드):**
```
공유/초대 링크 → 항상 웹에서 즉시 열림 (설치 장벽 0, K-factor 보호)
                └→ 결과 열람 후 "매일 아침 운세 받기" → 앱 설치 유도
앱 = 데일리 푸시 + 구독의 집 (리텐션·LTV)
웹 = 단건 결제 (수수료 ~3%) / 앱 = 구독 (IAP 15~30% 감수, LTV로 회수)
```

**차별화 포지션 (레퍼런스 리뷰 마이닝 결과로 확정하되, 가설):** "겁주지 않는 사주" — 경쟁 앱 1점 리뷰의 공통 불만(불안 조장→결제 유도, 매번 같은 말, 과도한 광고)의 정반대. 공포 소구 없는 단정형 직설 + 처방 중심.

## 1. MVP 범위

### In (v1.0 출시 범위)
- [웹+앱 공통] 대화체 입력 → 만세력 산출 → 역술 시퀀스 카드 리포트 (8장)
- [웹] 공유 OG 이미지, 궁합 초대 루프, 단건 결제(토스페이먼츠)
- [앱] 동일 리포트 + 데일리 운세 푸시(오늘의 한 줄) + 구독(IAP)
- 결과 재접근: URL 토큰(웹) / 로컬 저장+토큰(앱)

### Out (만들지 않는다)
- 회원가입/로그인 (앱도 기기 기반 익명 ID + 토큰. 계정은 v2)
- 채팅형 상담, 관상/손금(사진 입력), 성명학, 서양 점성술
- 관리자 대시보드 (Vercel Analytics + 결제사 콘솔 + Supabase 쿼리로 대체)

## 2. 아키텍처 — 모노레포

### 2.1 구조
```
/ (pnpm workspace + Turborepo)
├─ packages/
│   ├─ core/                 # ★ 프레임워크 독립 순수 TS. 웹·앱 공유
│   │   ├─ saju/             # 만세력 엔진 (섹션 3)
│   │   ├─ interpret/        # 해석 유닛 분해·프롬프트·가드레일 (섹션 4)
│   │   ├─ compat/           # 궁합 계산
│   │   └─ content/          # 정적 카피 (일간 10종, 일주 60종 훅, 처방 데이터)
│   ├─ ui/                   # 공유 디자인 토큰·카드 컴포넌트 스펙 (RN/웹 각자 구현하되 토큰 공유)
│   └─ api-client/           # 웹·앱 공용 API 타입·클라이언트
├─ apps/
│   ├─ web/                  # Next.js 15 (랜딩, 결과 SSR, OG, 궁합, 웹결제, API 라우트)
│   └─ mobile/               # Expo (React Native) — iOS/AOS
└─ tests/                    # 엔진 검증 케이스 (섹션 12)
```

### 2.2 스택 확정
| 레이어 | 선택 | 비고 |
|---|---|---|
| 모노레포 | pnpm + Turborepo | core 패키지를 웹·앱이 공유 |
| 웹 | Next.js 15 (App Router) + Tailwind | SSR 공유 링크·OG 담당 |
| 앱 | **Expo SDK 최신 + TypeScript + expo-router** | 단일 코드베이스로 iOS/AOS. EAS Build로 스토어 배포 |
| 서버 | Next.js API Routes (웹앱 공용 백엔드) | 앱도 이 API를 호출. 별도 서버 없음 |
| DB | Supabase (Postgres) | 결과·토큰·해석캐시·이벤트·결제기록 |
| LLM | Anthropic API | 무료=Haiku급, 유료=Sonnet급 |
| 공유 이미지 | @vercel/og | 토큰별 동적 생성 (1:1, 9:16) |
| 결제(웹) | 토스페이먼츠 단건 | 비회원 결제 |
| 결제(앱) | RevenueCat + StoreKit/Play Billing | 구독 관리 단순화 |
| 푸시 | Expo Notifications (+FCM/APNs) | 데일리 운세 발송은 Supabase cron + Edge Function |
| 딥링크 | Universal Links(iOS)/App Links(AOS) | 설치자→앱, 미설치자→웹 자동 분기 |
| 분석 | Vercel Analytics + 자체 이벤트 테이블 | 퍼널·K-factor 계측 |

### 2.3 앱-웹 분기 규칙
- 모든 공유 URL은 `https://{domain}/r/{token}` 형태 하나로 통일 → 딥링크가 앱 설치 여부로 분기.
- 궁합 초대 수신자는 **무조건 웹 경험** (설치 요구 금지). 궁합 결과 열람 후에만 앱 유도 배너.

## 3. 만세력 계산 엔진 (packages/core/saju)

> 서비스 신뢰의 전부. 검증 케이스(섹션 12) 통과 전 UI 착수 금지.

### 3.1 입출력
입력: `{ birthDate, birthTime?, gender?, calendarType: 'solar'|'lunar'|'lunarLeap', birthRegion?: RegionCode, unknownTime: boolean }`
출력 `SajuChart`: 연/월/일/시주(천간·지지, 한자+한글) · 오행 분포·강약 · 일간 기준 십신 · 대운 리스트(시작나이, 10년 간지) · 지장간 · 12운성

### 3.2 계산 규칙 (한국 표준)
1. 연주 경계 = **입춘 절입 시각** (1/1 아님)
2. 월주 경계 = 12절 절입 시각 (중기 아님)
3. 일주 = 60갑자 순환 (기준일 앵커 방식)
4. 시주 = 일간 기준 시두법, 2시간 단위 12지지
5. **진태양시 보정 — 출생지 경도 기준 정밀화 (v1.1 변경):** 시·도 선택값의 대표 경도로 보정분 계산 (예: 서울 126.98°E → 약 -32분, 부산 129.08°E → 약 -24분). 미선택 시 기본 -30분. 지역 코드→경도 테이블은 `core/saju/regions.ts`
6. 서머타임 보정: 한국 시행 기간(1948–1951, 1955–1960, 1987–1988) -1시간. 기간 테이블 하드코딩
7. 야자시/조자시: v1 기본 = 자정 기준 일주 변경(조자시설). 상수로 전환 가능하게
8. 출생시 모름: 3주 리포트 폴백 + 전용 카피 분기

### 3.3 구현 방침
- **1안(권장):** 절기 절입 시각 테이블(1900–2050) + 음양력 대조 테이블(윤달 포함)을 JSON 내장, 자체 계산. 외부 API 의존 제로.
- 2안(교차검증용): lunar-javascript 등 오픈소스 결과와 비교 스크립트 작성 (중국 표준시 기준 절기와 시차 주의).

## 4. AI 해석 레이어 (packages/core/interpret)

### 4.1 파이프라인
```
SajuChart → 해석 유닛 분해 → Supabase 캐시 조회 → (미스 시) LLM 생성 → 가드레일 필터 → 조립
```
- **LLM에 생년월일 원본 절대 미전달.** 간지·오행·십신 JSON만.
- 캐시 키 = `유닛종류:유닛값:프롬프트버전:시즌`

### 4.2 해석 유닛과 캐시 전략 (비용 디커플링)
| 유닛 | 조합 수 | 성격 |
|---|---|---|
| 일간 타입 캐릭터 | 10 | 정적 콘텐츠 (content/에 선작성) |
| 일주 단정형 훅 | 60 | **정적, 사람이 직접 작성 (최우선 카피 작업)** |
| 오행 밸런스 코멘트 | 수백 패턴 | LLM 생성 + 캐시 |
| 관심사×일간 하반기 운 | 4×10×연간지 | LLM 생성 + 캐시, 시즌 태그 |
| 개운 처방 | 조합 규칙 기반 | 규칙 테이블 + 문구 캐시 |
| 유료 문답 답변 | 사용자별 | 캐시 없음, Sonnet급 풀 생성 |
→ 무료 리포트는 출시 후 캐시 적중률 90%+ 수렴 → 바이럴 시 비용 폭탄 없음.

### 4.3 역술가 페르소나 & 화법 규칙 (프롬프트 공통부)
- 페르소나: 30년 경력, 다정하지만 돌려 말하지 않는 역술가. 존댓말이되 상담체("~하죠", "~거든요").
- **화법 구조: 단정 먼저, 위로 나중.** "당신은 리더십이 있는 편이에요"(금지) → "사람들이 당신한테 기대죠. 근데 정작 당신이 기댈 데는 없고."(권장)
- 근거 없는 단정 금지: 모든 단정은 원국 데이터(십신·오행) 또는 유저가 고른 관심사에 근거.
- 부정 요소는 반드시 "주의+대처" 프레임: "10월엔 큰 결정을 한 템포만 늦추세요."

### 4.4 가드레일 (프롬프트 명시 + guardrails.ts 후처리 이중 방어)
- 금지: 건강 단정(수술·질병·사고 예언), 투자 지시, 법률·의료 조언, 사망·이혼 단정, **공포 소구("액땜 안 하면", "큰 화를 입는다" 류)**, 부적·굿 등 추가 지출 유도 표현.
- 모든 리포트 하단 고지: "본 콘텐츠는 오락·자기이해 목적이며 의학·법률·투자 조언이 아닙니다."
- 프롬프트 버전 태깅 → 캐시 키 포함 (개선 시 자연 무효화).

## 5. 경험 설계 — 역술 시퀀스 (v1.1 신설, UI의 뼈대)

실제 철학관 상담 7단계를 디지털로 번역한 것. 카드 구성(섹션 6)은 이 시퀀스의 구현이다.

| 실제 상담 | 디지털 번역 | 구현 포인트 |
|---|---|---|
| ① 접수 문답 | 대화체 입력 폼 | "태어난 시간, 확실하세요?" / 모름 → "괜찮아요, 시 없이 보는 법도 있으니" |
| ② 팔자를 적는다 | **붓글씨 애니메이션** | 한지 질감 위에 여덟 글자가 붓으로 한 자씩 (2~3초 의도적 뜸 = 노동 착시). SVG 패스 애니메이션, 스킵 버튼 제공 |
| ③ 첫 마디 훅 | 단정형 훅 카드 | 일주 60종 전용 카피, 타이핑 연출 |
| ④ 반응 조율 | 원탭 반응 분기 | "어때요, 좀 맞나요?" [소름😳/반 정도/글쎄요] → 후반 화법 분기. 관심사 질문도 "요즘 뭐가 제일 답답해요?"로 |
| ⑤ 문답 | "복채 내고 하나 묻기" | 유료 킬러 기능. 고민 한 줄 → 원국+대운 기반 맞춤 답변. 결제 버튼 카피 = **"복채 내기"** |
| ⑥ 처방 | 개운 처방전 카드 | 색·방향·조심할 달·올해의 한 가지. 공유 카드 겸용 |
| ⑦ 다시 듣기 | 토큰 재방문 + (v2) TTS 음성 리포트 | 앱 리텐션과 연결 |

**반응 분기 로직:** [소름] → 후반 카드 더 깊은 단정 톤 / [글쎄요] → "겉모습과 속이 다른 사주라 그래요" 받아침 톤(실제 역술 화법). 분기는 해석 유닛의 톤 변형 캐시로 처리(조합 폭발 방지: 톤 2종만).

**비주얼 언어:** 한지 질감, 먹빛+주홍 낙관 포인트, 세리프(명조계) 헤드라인. 단, 올드하지 않게 — 여백 많은 모던 레이아웃 위에 전통 요소를 포인트로만. (Co-Star의 "신비주의를 미니멀로 푸는" 문법 참고)

## 6. 리포트 카드 구성 (v1.1 개편 — 8장)

| # | 카드 | 내용 | 설계 의도 |
|---|---|---|---|
| 0 | 팔자 연출 | 붓글씨 애니메이션 (카드 아님, 전환 씬) | 의식·기대감 |
| 1 | 단정형 훅 | 일주 60종 전용 단정 한 줄 + 타이핑 연출 | "어떻게 알았지" 순간 = 신뢰 확정 |
| 2 | 나의 팔자 | 4주 8자 + 오행 밸런스 차트(SVG 자체 구현) | 전문성 신호, 아는 유저 검증 포인트 |
| 3 | 타입 카드 | "나는 ○○ 타입" 일간 10종 캐릭터 (丙火=한낮의 태양 등) + 전용 일러스트 | MBTI식 자기 라벨 = 공유 동기 |
| — | 반응 체크 | "어때요, 맞나요?" 원탭 → 화법 분기 | 상호작용 감각 |
| 4 | 성격 코어 | 일주+십신 기반 3문장 + 키워드 태그 | 저장 유발 |
| — | 관심사 문답 | "요즘 뭐가 제일 답답해요?" [연애/커리어/재물/인간관계] | 개인화 분기 입력 (대화로 위장한 수집) |
| 5 | 2026 하반기 운 | 선택 관심사 중심 키워드 3개 + 한 줄 | 체감 적중률 증폭 (정당한 포러 효과) |
| 6 | 조심할 것 (티저) | 1개 공개 + 월별 상세 블러 | 유료 훅 |
| 7 | 개운 처방전 | 색/방향/조심할 달/올해의 한 가지 + 워터마크 | **공유 카드 겸용** (9:16 저장 버튼) |
| 8 | CTA | [복채 내고 하나 묻기] [궁합 보러가기] [매일 아침 운세(앱)] | 전환 허브 |

- 카드당 3초 소화 가능 정보량. 세로 스와이프(앱)/스와이프+클릭(웹).
- 일간 10종 일러스트: 생성형 이미지로 초안 → 통일 리터치 → 정적 자산. 실시간 이미지 생성 API 금지.

## 7. 입력 설계 — 프로그레시브 프로파일링 (v1.1 신설)

원칙: **선최소 후수집.** 첫 리포트 전 입력은 4필드·30초 이내. 이후 단계마다 대화로 위장해 하나씩.

```
[무료 진입]   생년월일 + 출생시(모름 가능) + 성별(선택) + 출생지역(시·도)  ← 지역은 경도 보정 기능 입력
[리포트 중간] 반응 체크(원탭) / 관심사 1택(원탭)                        ← 화법·콘텐츠 분기
[유료 결제 후] 현재 상태(직장인/취준/자영업/학생 · 솔로/연애/썸) + 고민 한 줄  ← 풀 개인화
[선택·재미]   MBTI → "사주×MBTI" 크로스 카드 (공유 변주 소재)
[앱 전환 후]  푸시 허용 → 데일리 운세. (v2) 데일리 기분 체크인
```

**보류 확정 (v1에서 만들지 않음):** 이름(성명학), 얼굴 사진(관상)·손금 — 생체정보 리스크 + 이미지 비용이 트래픽 비례(디커플링 파괴) + 심사 리스크. 데일리 체크인은 v2.

## 10. 앱 전략 (v1.1 신설)

### 10.1 앱의 존재 이유 = 리텐션

### 10.2 스토어 심사 대응
- **애플 4.3(스팸/템플릿 앱) 리스크:** 운세 단일 기능 앱은 반려 사례 有 → 심사 서류·스크린샷에서 "정밀 만세력 계산기(출생지 경도 보정) + 개인화 리포트" 유틸리티 실체 강조. 붓글씨 연출 등 고유 UX가 방어 자산.
- 미신 조장 심의: 오락 목적 고지문을 앱 내 상시 노출. 연령 등급 설정 확인.
- 구글: 결제 정책(구독 IAP) 준수 외 특이 리스크 낮음.

### 10.3 배포 파이프라인
- EAS Build + EAS Submit. 내부 테스트: TestFlight / Play 내부 테스트 트랙 → 클로즈드 베타 30명.
